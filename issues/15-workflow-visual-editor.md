## Description
Workflows are currently defined in YAML files. We need a visual drag-and-drop editor that lets users build workflows without writing YAML.

## Requirements

### Visual Editor Features
- Canvas with nodes and connections
- Drag agents from palette onto canvas
- Connect agent outputs to agent inputs
- Configure each step inline (task, parameters, gates)
- Real-time validation (detect cycles, missing inputs)
- Export to YAML / Import from YAML

### Node Types
```
[Start] → [Agent Step] → [Gate] → [Agent Step] → [End]
                                     ↓ (branch)
                                  [Parallel Step]
```

### Agent Palette
Sidebar showing all available agents with:
- Agent name and icon
- Tool list (hover to expand)
- Current status (online/offline/busy)
- Provider info

### Step Configuration Panel
When a node is selected:
- Task description (textarea with template variables)
- Agent selector (dropdown with filtering)
- Provider override (optional)
- Gates configuration (human approval, conditions, cost checks)
- Expected output type

### Output
```yaml
# Generated by visual editor
workflow:
  id: my-workflow
  steps:
    - id: step1
      agent: researcher
      task: "Research {{ topic }}"
```

## Acceptance Criteria
- [ ] Canvas renders with drag-and-drop support
- [ ] Agents can be dragged from palette to canvas
- [ ] Connections between nodes are drawn correctly
- [ ] Configuration panel opens when node is clicked
- [ ] YAML export produces valid workflow files
- [ ] YAML import correctly renders existing workflows
- [ ] Cycle detection prevents invalid connections
- [ ] Responsive design works on desktop

## Additional Context
This makes Chakravyuh accessible to non-developers who want to build AI workflows without writing code.
